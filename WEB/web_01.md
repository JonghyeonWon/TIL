## 웹 서비스 구성 요소

- __사용자 (Client):__ 브라우저, 모바일 앱 등 사용자가 직접 접하는 영역
- __프론트엔드 (FE):__ 일종의 UI 역할, 사용자의 요청을 백엔드에 전달하고 백엔드로부터 받은 응답을 사용자에게 보여줌
- __백엔드 (BE):__ 전달받은 요청을 처리하는 영역으로 비즈니스 로직이 이 단계에 포함됨
- __데이터베이스 (DB):__ 리소스를 영속적으로 저장하는 물리적인 영역

__핵심 통신 규칙__: 사용자는 오직 프론트엔드와만 소통하며, 백엔드 및 DB와 직접적으로 소통하는 것은 불가능함

---

## 웹 서비스의 기본 동작 흐름 

웹 서비스는 사용자(클라이언트)와 서버가 __요청(Request)__ 과 __응답(Response)__ 을 주고받으며 동작함

- __전체 처리 Flow__

1) [사용자 이벤트 발생] 
2) [Request가 서버로 전달] 
3) [서버가 Event 처리] 
4) [Response 반환 (UI 출력)]

## API & DevOps
__API (Application Programming Interface)__
- 프론트엔드와 백엔드 사이의 통신 규칙
- 주로 Swagger 같은 API 명세를 통해 소통

__DevOps__: 개발과 시스템 운영을 병행하는 개발
